# 내부 링크 & 트래픽 구조

검색엔진은 링크를 따라 사이트를 이해합니다. 글이 서로 고립되어 있으면(=오펀 페이지)
아무리 잘 써도 상위 노출이 어렵습니다. 이 문서는 서기(현)가 사용하는
**토픽 클러스터(topic cluster)** 방식의 내부 링크 구조를 설명합니다.

## 1. 토픽 클러스터 구조

카테고리(폴더)는 콘텐츠를 "저장"하는 단위이고, 클러스터는 "주제"로 묶는 단위입니다.
같은 클러스터에 속한 글은 카테고리가 달라도 서로 연결되어야 합니다.

```
                 ┌──────────────────────┐
                 │  허브: /study (철도기술사) │
                 └──────────┬───────────┘
                            │
        ┌───────────────────┼───────────────────┐
        │                    │                    │
 클러스터: 선형/캔트      클러스터: 궤도/틀림      클러스터: 도상
        │                    │                    │
 study/transition-curve    study/track-irregularity  standard/rail-fastening-system
 terms/cant                news/track-inspection...  case/concrete-track-construction
 standard/minimum-curve-radius
```

- **허브 페이지**: `/study`, `/terms`, `/standard`, `/case`, `/resource`, `/news` 6개
  카테고리 목록 페이지. 해당 카테고리의 모든 글로 나가는 링크를 모아두는 진입점입니다.
- **클러스터**: `tags`로 묶이는 주제 그룹. 카테고리를 가로지릅니다.
  예를 들어 "궤도" 클러스터는 `study`(원리 설명) + `terms`(용어 정의) +
  `standard`(설계기준) + `news`(관련 뉴스)를 전부 포함할 수 있습니다.
- **스포크(spoke)**: 클러스터에 속한 개별 글. 서로 링크를 주고받아야 합니다.

## 2. 관련 글 추천 알고리즘 (자동)

`src/lib/posts.ts`의 `getRelatedPosts`가 클러스터 연결을 자동으로 수행합니다.

```ts
score = tagOverlap * 2 + (sameCategory ? 1 : 0)
```

- **태그 겹침이 카테고리 일치보다 가중치가 높습니다.** 즉, 다른 카테고리라도 태그가 많이
  겹치면 같은 카테고리의 태그 없는 글보다 먼저 추천됩니다. → 카테고리 사일로(silo)에 갇히지 않고
  클러스터 단위로 연결됩니다.
- 태그 겹침이 하나도 없으면 같은 카테고리 최신글로 자동 채워집니다(항상 "관련 글" 섹션이
  비어있지 않도록 하는 안전장치).
- **결론**: 관련 글이 잘 뜨게 하려면 새 글을 쓸 때 기존 글과 **태그를 일관되게 재사용**하세요.
  새 태그를 만들기 전에 [content/README.md](../content/README.md)의 기존 태그 목록을 먼저 확인하는 습관을 들이면 좋습니다.

## 3. 본문 내 상호링크 작성 규칙 (수동)

자동 추천(`관련 글` 섹션)과 별개로, **본문 안에서 직접 링크를 거는 것**이 SEO에는 훨씬 강한 신호입니다.

- **신규 글 작성 시**: 관련된 기존 글을 최소 2개, 본문 문맥 안에서 자연스럽게 링크합니다.
  ```mdx
  캔트에 대한 자세한 설명은 [캔트 (Cant)](/terms/cant)를 참고하세요.
  ```
- **허브 → 스포크**: 카테고리 목록 페이지(허브)는 이미 모든 글로 링크되어 있으니 추가 작업이 필요 없습니다.
- **스포크 → 허브**: 개별 글 상세 페이지에는 브레드크럼(`홈 > 카테고리 > 글 제목`)이 항상 노출되어
  자동으로 허브로 돌아가는 링크가 생성됩니다.
- **오래된 글 보강**: 새 글을 쓸 때, 관련된 과거 글 본문에도 새 글로 가는 링크를 1개 추가하는 것을
  권장합니다(양방향 링크). 특히 `terms`(용어 정의)는 다른 모든 글에서 인용되기 좋은 "링크 허브" 역할을
  하므로, `study`/`standard`/`case`/`news` 글 본문에서 관련 용어가 언급되면 해당 `terms` 글로 링크하세요.

## 4. 카테고리 허브 페이지 SEO

각 카테고리 목록 페이지는 다음 구조화 데이터/메타를 이미 갖추고 있습니다 (`src/app/{category}/page.tsx`).

- `CategoryPageHeader`: 카테고리 소개문 (검색엔진이 허브 페이지의 주제를 이해하는 텍스트 신호)
- `BreadcrumbJsonLd`: `홈 > 카테고리` 구조화 데이터
- `CollectionPageJsonLd`: 해당 카테고리의 전체 글 목록을 `ItemList`로 구조화 — 검색엔진이
  허브 페이지를 "글 모음 페이지"로 명확히 인식하게 합니다.
- `openGraph`/`twitter` 메타 (Navy 톤의 OG 이미지 포함)

새 카테고리를 추가하지 않는 한 이 구조는 코드 수정 없이 새 글이 추가될 때마다 자동으로
최신 상태를 반영합니다.
